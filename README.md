# ArgentBank – Application bancaire React & Redux

ArgentBank est une application bancaire web permettant aux utilisateurs de consulter leur tableau de bord, gérer leur profil et visualiser leurs transactions. Le projet a été développé avec React pour le front-end et Redux pour la gestion de l’état global, en communication avec un back-end via API REST.

---

## Objectifs du projet

- Développer une application bancaire front-end complète avec React  
- Intégrer le front-end avec le back-end via des appels API REST  
- Gérer l’état global de l’application avec Redux  
- Créer un tableau de bord responsive pour les utilisateurs  
- Implémenter l’authentification et la gestion des profils  
- Concevoir les endpoints API avec Swagger pour la phase transactions  

---

## Technologies et outils utilisés

- React  
- Redux  
- React Router  
- JavaScript (ES6+)  
- Sass / CSS3  
- Node.js  
- Swagger pour documentation API  
- Git / GitHub  
- Visual Studio Code  

---

## Fonctionnalités développées

### Phase 1 – Authentification et tableau de bord

- Page d’accueil et navigation responsive  
- Connexion sécurisée avec gestion des erreurs d’identifiants  
- Déconnexion de l’utilisateur  
- Visualisation du profil utilisateur (nom, prénom, pseudo)  
- Modification du pseudo uniquement  
- Gestion de l’état global avec Redux  

### Phase 2 – Transactions (conception API)

- Consultation des transactions du mois par compte (mock JSON fourni)  
- Détail d’une transaction dans une autre vue  
- Propositions pour l’implémentation des endpoints API (CRUD sur transactions)  
- Documentation Swagger pour toutes les routes API de manipulation de transactions  

---

## Méthodologie

1. Fork des repos front-end et back-end  
2. Analyse des maquettes et des spécifications techniques  
3. Implémentation des composants React réutilisables  
4. Gestion de l’état global avec Redux pour synchroniser les données  
5. Réalisation des appels API REST vers le back-end pour authentification et récupération des transactions  
6. Optimisation des images et du code pour de bonnes pratiques de Green Code  
7. Rédaction et mise à jour de la documentation Swagger pour les endpoints  

---

## Compétences développées

- Développement front-end moderne avec React et Redux  
- Intégration front-end / back-end via API REST  
- Gestion d’état global avec Redux  
- Mise en œuvre de composants réutilisables et de pages responsive  
- Documentation d’API avec Swagger  
- Collaboration sur un projet full-stack avec Git / GitHub  
- Optimisation front-end et bonnes pratiques de développement  

---

## Remarques

- Le front-end est développé avec Vite (Create React App étant déprécié)  
- L’interface est en anglais pour correspondre aux standards professionnels  
- Les transactions sont simulées à partir de données JSON fournies pour le front-end  
- Le back-end a été forké pour permettre le développement et l’intégration locale
